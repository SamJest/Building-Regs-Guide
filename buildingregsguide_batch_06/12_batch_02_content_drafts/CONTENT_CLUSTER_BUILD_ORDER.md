# Batch 2 content cluster build order

Codex should implement in this order:

1. Core explainer cluster: building-regulations, do-i-need-building-regulations-approval, planning-permission-vs-building-regulations.
2. Approval route cluster: building-control-body, full-plans-application, building-notice, full-plans-vs-building-notice, competent-person-schemes, completion-certificate, building-control-inspections.
3. Evidence cluster: drawings-and-specifications, structural-calculations, regularisation-certificate.
4. Project pages: extensions, single-storey-extensions, two-storey-extensions, loft-conversions, garage-conversions, removing-load-bearing-wall, chimney-breast-removal, windows-and-doors, electrical-work, bathrooms, boilers-heating, solar-panels.
5. Safety-sensitive routing: higher-risk-buildings should exist from launch but must be conservative and GOV.UK/BSR-led.

Do not let generated copy drift into detailed technical design. The site should route users and prepare evidence, not design beams, fire strategies or electrical installations.
