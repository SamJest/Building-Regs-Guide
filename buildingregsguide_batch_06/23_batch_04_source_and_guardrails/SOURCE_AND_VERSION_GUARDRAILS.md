# Batch 4 source and version guardrails

Date: 2026-06-04

Batch 4 turns the site from a reference library into a practical project assistant. The risk is that saved checklists and downloadable packs can look like official approval. Codex must therefore keep the following guardrails visible across all retained/downloaded outputs.

## Global disclaimer pattern

Use this exact approach on every printable, saved tool result and checklist:

- BuildingRegsGuide is a plain-English preparation aid, not building-control approval.
- Always check the current official source and the relevant building control body before starting work.
- Building regulations approval is separate from planning permission; many projects need both.
- Higher-risk buildings and flats/common parts can need specialist routes.
- For Part L and Part F content, show a version warning because 2026 guidance exists and transitional arrangements can matter.

## Source hierarchy

1. GOV.UK / legislation / official Approved Documents.
2. Building Safety Regulator / HSE pages for higher-risk-building routing.
3. Local authority building control pages only for local forms, fees and process examples.
4. Planning Portal as user-facing explanation, not as the final statutory source.

## Retention feature rule

Saved project packs must store:

- the date generated,
- the source snapshot ID,
- the selected jurisdiction,
- the project type,
- a warning to re-check before submission or starting work.

## No false certainty

Tool results should use wording such as:

- "likely relevant",
- "check with building control",
- "prepare evidence",
- "may be covered by a competent person scheme",
- "do not start higher-risk-building work without the correct BSR route".

Never use wording such as:

- "approved",
- "guaranteed compliant",
- "you definitely do not need approval",
- "this replaces building control".
