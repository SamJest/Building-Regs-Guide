# Tool result component contract

Every building-regs tool result must render the same component structure:

1. Result headline
2. Confidence label: `routing aid`, `likely`, `needs professional check`, or `higher-risk route`
3. Why this result appeared
4. What to do before starting
5. Evidence checklist
6. Inspection/sign-off checklist
7. Approved Documents likely to matter
8. Separate permissions warning
9. Source links and last checked date

Never render a result that says simply `approved`, `not approved`, `legal`, or `safe to start`.
