# Updated total batch roadmap

Six batches is the cleanest target. More than that risks over-planning; fewer would leave too much for Codex.

## Batch 1 - Foundation
Strategy, site positioning, route registry, source registry, tool registry, download registry, page briefs, Codex master prompt.

## Batch 2 - Core content and tool logic
Core page drafts, project requirement matrix, inspection-stage library, competent-person data, first JS decision engine, tool contracts and download asset specs.

## Batch 3 - Approved Documents and project depth
Deepen Approved Document hub pages, project-specific pages, “what applies to my project” logic, and source update rules.

## Batch 4 - Retention and assets
Printable PDFs, evidence folders, checklists, calculators, save/share/email flows and “next project” journeys.

## Batch 5 - SEO and scale
Internal linking graph, schema strategy, sitemap rules, long-tail route generation, council/local page rules, anti-thin-content validation.

## Batch 6 - Final Codex handover and launch QA
One master execution prompt, staged Codex prompts, validation scripts, launch checklist, analytics/GSC setup and post-launch roadmap.
