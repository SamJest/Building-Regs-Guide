# Monetisation and Retention Later

Do not optimise Batch 1 around immediate monetisation.

For the first build, prioritise:

- trust
- official-source grounding
- useful tools
- printable assets
- return visits through saved project files/checklists
- clean internal linking
- Search Console learning

Later monetisation options:

- display ads once traffic is meaningful
- low-friction sponsored placement only on relevant pages
- building control professional / structural engineer lead routes once trust exists
- affiliate/referral only where useful and disclosed
- paid printable packs only after free resources prove demand

Keep any early contact form secondary. The user should get a useful result before being asked for details.
