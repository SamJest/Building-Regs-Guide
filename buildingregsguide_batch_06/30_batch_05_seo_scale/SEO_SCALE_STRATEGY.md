# Batch 5 SEO scale strategy

## Objective

Build a scalable but safe route system for BuildingRegsGuide. The site should not become a pile of thin pages. It should become a building-control decision layer with source-backed pages, tools, checklists and saved project workflows.

## What Batch 5 adds

- 259 new candidate route records.
- 393 combined candidate route records.
- Page-family templates for project questions, Approved Document long-tail pages, download landings, comparison pages and evidence/certificate pages.
- Schema templates and JavaScript generators.
- Internal linking graph and contextual linking rules.
- Sitemap, canonical and noindex rules.
- Validation scripts for Codex to wire into the build.
- Search/discovery data models.

## Core rule

Do not publish programmatic pages just because a route exists. A route is eligible only when it has:

1. A valid official source block.
2. A clear parent hub or project page.
3. At least 5 useful internal links.
4. A unique answer angle, not copied boilerplate.
5. A next step: tool, download, dashboard save, or related project.
6. Canonical URL and breadcrumb trail.
7. No unsupported legal certainty claims.

## Best launch order

1. Build core pages and existing Batch 1-4 content first.
2. Implement schema and validation helpers.
3. Publish comparison/evidence routes next because they are high intent and easier to make unique.
4. Publish download landing pages with real previews.
5. Publish question pages in waves of 20-30 after validation.
6. Keep local pages as a single useful gateway unless a local-data layer is added later.

## Sister-site relationship

UKPlanningGuide should send users to BuildingRegsGuide when the user has moved from “can I get planning permission?” to “what proof, approval, inspections and certificates do I need?” BuildingRegsGuide should send users back to UKPlanningGuide when the issue is planning permission, permitted development, prior approval, lawful development certificates or local planning policy.
